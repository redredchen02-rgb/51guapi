# 吃瓜小幫手 v0.2 Backlog

v0.1 已上線（多渠道爬取 + AI 提煉 + 導出）。以下為 v0.1 執行期記錄的延後項，按優先級排列。

## 安全加固（來自 U6 安全複查，P1/P2 殘留）

- [ ] **DNS rebinding TOCTOU**（P2）：`assertUrlSafe` 與底層 fetch 各解析一次 DNS，無 socket-level IP pinning。攻擊者可在「入庫校驗通過」後改 A 記錄指向私網。建議上 undici custom-lookup dispatcher，把解析到的公網 IP pin 到實際連線。
- [ ] **confirm 手勢強度**（P2）：`x-operator-confirm` header + body `confirm:true` 只防誤觸，不防越權——任何持 JWT 的調用方（含 XSS 竊 token）加兩個常量即可寫 allowlist。考慮二次驗證/簽名或限制寫入來源。
- [ ] **IDN 同形覆蓋**（P2）：`isMixedScript` 目前覆蓋拉丁+西里爾/希臘混用；純單一非拉丁脚本的同形 label（如切羅基 Ꭺ、亞美尼亞 օ）未攔。建議引 Unicode confusables 表或限制 label 為純單脚本。

## 功能延後

- [x] **CSV 導出**（done 2026-06-17，PR #1）：待審池批量導出 CSV，元資料 + 8 個吃瓜事實欄、正確轉義。
- [x] **遞歸多頁爬取 + `max_depth` 落地**（done 2026-06-17，PR #1）：`fetchListPaged` 跟隨翻頁，`channel.maxDepth` 端到端生效（1..50 clamp + `MAX_PAGES=50` 硬上限），`max_depth` 不再 reserved。

## 清理

- [ ] **內部包名去 publisher**：`publisher-monorepo` / `publisher-backend` / `publisher-fill-assistant`（非 scope、非用戶可見的 pnpm 標識符）仍含 "publisher"。改動會牽動 CI/腳本/命令裡的 `--filter` 引用，價值低但為求一致可一併清。
- [ ] **backend Docker 打包**：v0.1 已從 release.yml 移除 Docker 步驟。Dockerfile builder 階段已修（`--ignore-scripts` + 強制 shared emit），但 runtime 階段 `npm install --omit=dev` 不認 `@51guapi/shared` 的 `workspace:*` 協議（`EUNSUPPORTEDPROTOCOL`）。改用 `pnpm deploy` 產出扁平 node_modules，或從 builder 複製 prod 依賴 + 把 shared 放進 `node_modules/@51guapi/shared`。需在有 Docker daemon 的環境本地驗證後再接回 release.yml。
- [ ] **CI runner Node.js 升級**：`actions/checkout@v4`、`gitleaks-action@v2` 仍跑 Node.js 20（2026-09 將移除）。升級到支援 Node.js 24 的版本。
